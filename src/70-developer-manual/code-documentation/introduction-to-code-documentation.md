# Code Documentation
This chapter outlines recommended practices for adding inline documentation to ROS nodes and Python libraries developed within Duckietown projects. 

The guidelines detail **what** information should be recorded and **how** it should be formatted. Guidance for transforming these comments into a reader‑friendly website is provided in [](sec:dt_way_build_docs).
